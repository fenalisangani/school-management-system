# School Management System (Django)

Five integrated modules for academic institution management:

| Module | App | Features |
|--------|-----|----------|
| Class | `classes` | Academic years, classes, sections, subjects, allocations |
| Student Registration | `students` | Profiles, enrollment, documents |
| Teacher | `teachers` | Faculty profiles, class/subject assignments |
| Fees | `fees` | Structures, assignments, payments, reports (INR) |
| Attendance | `attendance` | Student/teacher attendance, bulk marking, analytics |

## Share on your local network (other devices, same Wi-Fi)

1. Find your PC's IPv4 address: `ipconfig | findstr /i "IPv4"`
2. Run: `py manage.py runserver 0.0.0.0:8000`
3. On another device open: `http://<YOUR-IP>:8000/`

Both devices must be on the same network. Allow Python through Windows Firewall if prompted.

## Share instantly over the internet (no install for viewers)

## Deploy permanently (free hosting, always online)

The project is deployment-ready (WhiteNoise static serving, gunicorn, env-based
settings, optional Postgres). Files included: `Procfile`, `render.yaml`,
`build.sh`, `runtime.txt`.

**Render.com (recommended, free):**
1. Push this folder to a GitHub repository.
2. On https://render.com â†’ **New â†’ Blueprint** â†’ connect the repo (it reads `render.yaml`).
3. (Optional) Add a free PostgreSQL database and set its `DATABASE_URL` env var so data persists.
4. Deploy. You get a permanent `https://<your-app>.onrender.com` URL anyone can open.

Required environment variables on the host:

| Variable | Value | Notes |
|----------|-------|-------|
| `SECRET_KEY` | (random string) | Auto-generated by `render.yaml` |
| `DEBUG` | `False` | Production |
| `ALLOWED_HOSTS` | your host domain | Auto-detected on Render |
| `DATABASE_URL` | Postgres URL | Optional; without it, SQLite is used |
| `CSRF_TRUSTED_ORIGINS` | `https://your-app.onrender.com` | Auto-detected on Render |

After deploy, create an admin user from the host shell: `python manage.py createsuperuser`.
